Implementation requirements per locked decisions:

**GET /me** (D-01, D-02, D-05):
- preHandler: `[requireRole("player")]` — returns 403 for team/admin, 401 for unauthenticated
- Query: `fastify.db.query.players.findFirst({ where: eq(players.userId, request.session!.user.id) })`
- If null: `return reply.status(404).send({ error: { code: "NOT_FOUND", message: "Profile not found" } })`
- If found: `return ok(profile)`

**PUT /me** (D-05, D-08, D-09):
- preHandler: `[requireRole("player")]`
- body schema: `UpsertPlayerProfileRequestSchema`
- Upsert using `.onConflictDoUpdate()` on `players.userId`
- INSERT values: `{ id: nanoid(), userId: request.session!.user.id, ...request.body, createdAt: new Date(), updatedAt: new Date() }`
- UPDATE set: `{ ...request.body, updatedAt: new Date() }` — CRITICAL: must set `updatedAt` explicitly (D-09)
- Map field names: `photoUrl → photoUrl`, `positions → positions`, `birthDate → birthDate`, `phone → phone`, `availability → availability`
- Return: `return ok(result)` where result is the `.returning()[0]`

**GET /:id** (D-03, D-07):
- preHandler: `[requireSession]` — any authenticated role permitted (D-07)
- params schema: `z.object({ id: z.string() })`
- Query: `fastify.db.query.players.findFirst({ where: eq(players.id, request.params.id) })`
- If null: `return reply.status(404).send({ error: { code: "NOT_FOUND", message: "Profile not found" } })`
- If found: `return ok(profile)`

Use `fastify.withTypeProvider<ZodTypeProvider>()` for each route that has a schema. Import `ZodTypeProvider` from `fastify-type-provider-zod`. Import `eq` from `drizzle-orm`. Import `nanoid` from `nanoid`.

The plugin function signature: `const playersRoutes: FastifyPluginAsync = async (fastify) => { ... }`

Default export: `export default playersRoutes`

**Important:** The `players` DB table stores `positions` and `skills` as `text[]` (Postgres arrays). The contract sends/receives `string[]`. Drizzle handles this mapping transparently — no manual conversion needed. However, `height` and `weight` in the DB are `integer` but the contract sends them as numbers — pass through directly.

**Date serialization:** The DB returns `Date` objects for `createdAt`/`updatedAt` but the contract expects ISO datetime strings. Either:
- Cast the returned profile to match the contract: `{ ...profile, createdAt: profile.createdAt.toISOString(), updatedAt: profile.updatedAt.toISOString() }`
- Or verify if Drizzle serializes Date → ISO string automatically with `withTimezone: true` — if tests pass without manual conversion, leave it
