    5. `src/config/env.ts`: Create a Fastify plugin using `fastify-plugin` that validates env vars with Zod. Schema:
       - `NODE_ENV`: z.enum(["development", "production", "test"]).default("development")
       - `PORT`: z.coerce.number().default(3000)
       - `DATABASE_URL`: z.string().min(1) (required, no default)
       - `JWT_SECRET`: z.string().min(32) (required, no default)
       - `CORS_ORIGIN`: z.string().default("http://localhost:5173")

       Use `@fastify/env` with JSON Schema converted from Zod, OR manually parse `process.env` with Zod and decorate. The plugin must:
       - Wrap with `fp()` from `fastify-plugin` so `fastify.config` is globally visible
       - Throw a descriptive error if validation fails (fail-fast)
       - Augment `FastifyInstance` with `config: Env`

    6. `src/plugins/cors.ts`: Register `@fastify/cors` with:
       ```typescript
       origin: fastify.config.CORS_ORIGIN,
       credentials: true,
       ```
       Wrap with `fp()`.

    7. `src/routes/health.ts`: GET "/" handler that returns `{ status: "ok" }`. Do NOT wrap with `fp()` (route plugins are scoped).

    8. `src/lib/response.ts`: Two exported functions:
       ```typescript
       export const ok = <T>(data: T) => ({ data })
       export const list = <T>(data: T[], page: number, pageSize: number, total: number) => ({
         data,
         meta: { page, pageSize, total, totalPages: Math.ceil(total / pageSize) },
       })
       ```

    9. `src/lib/errors.ts`: Custom `AppError` class extending `Error` with `statusCode` and `code` fields. Register a Fastify `setErrorHandler` that:
       - Returns `{ error: { code, message, details? } }` matching the frontend `ApiErrorSchema`
       - Handles `AppError` instances with their status code
       - Handles Zod validation errors with 400
       - Falls back to 500 for unknown errors

    10. `src/app.ts`: Factory function `buildApp()`:
        ```typescript
        export async function buildApp() {
          const fastify = Fastify({
            logger: process.env.NODE_ENV !== "test",
          }).withTypeProvider<ZodTypeProvider>()
          fastify.setValidatorCompiler(validatorCompiler)
          fastify.setSerializerCompiler(serializerCompiler)
          await fastify.register(import("./config/env.js"))
          await fastify.register(import("./plugins/cors.js"))
          await fastify.register(import("@fastify/sensible"))
          // Error handler from lib/errors.ts
          await fastify.register(import("./routes/health.js"), { prefix: "/health" })
          return fastify
        }
        ```
        Note: Do NOT register the DB plugin here yet — Plan 03 wires it after the schema is built.

    11. `src/server.ts`: Import `buildApp`, call it, then `fastify.listen({ port: fastify.config.PORT, host: "0.0.0.0" })`. Catch errors and `process.exit(1)`.
