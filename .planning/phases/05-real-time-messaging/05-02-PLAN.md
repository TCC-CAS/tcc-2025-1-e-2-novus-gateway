    **Endpoint 1 — POST /conversations (MSG-01, D-09, D-10):**
    - preHandler: [requireSession]
    - body schema: `z.object({ participantId: z.string() })`
    - Get userId from request.session!.user.id
    - Look up user's current subscription plan from `subscriptions` table (same pattern as subscription.ts)
    - Get plan limits: `getPlanLimits(subscription.planId, subscription.role)`
    - Count existing conversations: `SELECT COUNT(*) FROM conversations WHERE participantA = userId OR participantB = userId`
    - If count >= conversationsLimit → throw forbidden("Conversation limit reached for your plan")
    - Normalize participant order (per D-04 unique_participants constraint): `const [pA, pB] = [userId, participantId].sort()`
    - Upsert (atomic, per D-10): `db.insert(conversations).values({ id: nanoid(), participantA: pA, participantB: pB, createdAt: new Date(), updatedAt: new Date() }).onConflictDoUpdate({ target: [conversations.participantA, conversations.participantB], set: { updatedAt: new Date() } }).returning()`
    - If inserted (createdAt === updatedAt within ~1ms): reply.code(201).send(ok(conversation))
    - Else: reply.code(200).send(ok(conversation))  // idempotent return

    **Endpoint 2 — GET /conversations (MSG-01, D-14):**
    - preHandler: [requireSession]
    - No query schema needed (return all user's conversations)
    - Query: SELECT conversations.* plus participant user info plus unreadCount subquery:
      ```typescript
      const rows = await fastify.db
        .select({
          id: conversations.id,
          participantA: conversations.participantA,
          participantB: conversations.participantB,
          updatedAt: conversations.updatedAt,
          createdAt: conversations.createdAt,
        })
        .from(conversations)
        .where(
          or(
            eq(conversations.participantA, userId),
            eq(conversations.participantB, userId)
          )
        )
        .orderBy(desc(conversations.updatedAt))
      ```
    - For each conversation, compute:
      - participantId = (row.participantA === userId) ? row.participantB : row.participantA
      - Fetch participant user: `db.select().from(users).where(eq(users.id, participantId))`
      - unreadCount: `SELECT COUNT(*) FROM messages WHERE conversationId = row.id AND senderId != userId AND readAt IS NULL` (per D-14)
      - lastMessage: `SELECT * FROM messages WHERE conversationId = row.id ORDER BY id DESC LIMIT 1`
    - Build ConversationSummary array and return with meta: `{ page: 1, pageSize: rows.length, total: rows.length, totalPages: 1 }`

    **Endpoint 3 — GET /conversations/:id/messages (MSG-02, D-12, D-13, D-15):**
    - preHandler: [requireSession]
    - params schema: `z.object({ id: z.string() })`
    - Verify conversation exists and user is a participant: query conversations table, if not found throw notFound()
    - Auto-mark as read (per D-15): `UPDATE messages SET readAt = now() WHERE conversationId = id AND senderId != userId AND readAt IS NULL`
    - Fetch all messages ordered ASC: `SELECT * FROM messages WHERE conversationId = id ORDER BY id ASC`
    - Return with meta: `{ page: 1, pageSize: msgs.length, total: msgs.length, totalPages: 1 }`

    **Endpoint 4 — POST /conversations/:id/messages (MSG-02, D-05):**
    - preHandler: [requireSession]
    - params schema: `z.object({ id: z.string() })`
    - body schema: `z.object({ content: z.string().min(1).max(2000) })`
    - Verify conversation exists and user is a participant — throw notFound() if not
    - Insert message: `db.insert(messages).values({ conversationId: id, senderId: userId, content, createdAt: new Date() }).returning()`
      Note: messages.id is bigserial — do NOT provide id value, let DB auto-generate
    - Emit Socket.io event AFTER DB save (per D-05 — HTTP source of truth):
      `fastify.io.to(id).emit("new_message", message)`
      Note: `fastify.io` is decorated by the socket-io plugin registered in Plan 03. If plugin not yet
      registered, this call is a no-op — test the HTTP path works first; socket delivery tested in Plan 03.
    - Return reply.code(201).send(ok(message))
